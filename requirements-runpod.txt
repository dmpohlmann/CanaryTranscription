# GPU/RunPod dependencies for CanaryTranscription (transcribe_runpod.py).
# For the official "RunPod PyTorch 2.x" template, which already ships torch +
# CUDA — so torch is intentionally NOT listed here.
#
# Quick setup (creates a venv + HF cache on the persistent network volume):
#   bash setup-runpod.sh
#   source /workspace/venv/bin/activate
#   python transcribe_runpod.py --url <audio-url>
#
# Or manually:
#   apt-get update && apt-get install -y ffmpeg
#   python -m venv --system-site-packages /workspace/venv   # inherit image torch
#   /workspace/venv/bin/pip install -r requirements-runpod.txt
#   export HF_HOME=/workspace/hf                             # persist model cache
#
# Notes:
#  * flash-attn is intentionally omitted — transcribe_runpod.py uses sdpa
#    attention (word-level timestamps need cross-attention weights flash-attn
#    can't return), and flash-attn triggers a long, often-failing source build.
#  * torch/torchvision/torchaudio are intentionally omitted — the RunPod
#    PyTorch image already ships a CONSISTENT, cu124-matched trio
#    (torch 2.4.1 / torchvision 0.19.1 / torchaudio 2.4.1). The venv inherits
#    them via --system-site-packages. Do NOT install or upgrade them: bumping
#    torch alone strands the image's torchvision/torchaudio and triggers
#    "operator torchvision::nms does not exist" on `import transformers`.
#  * pyannote.audio is pinned to <4 ON PURPOSE. pyannote.audio 4.x requires
#    torch>=2.8, which would force exactly that torch upgrade. The 3.x line
#    works with the image's torch 2.4.1, and transcribe_runpod.py already
#    supports the pyannote 3.x API (the use_auth_token fallback).
#  * ffmpeg is a system package (apt), used by librosa to decode M4A/AAC.

transformers
accelerate
librosa
pyannote.audio<4     # 4.x needs torch>=2.8; staying on 3.x keeps the image's torch 2.4.1
punctuators          # on-device punctuation + truecasing for --restore-punct
